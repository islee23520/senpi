- Pre-fix focused run failed with:
  AssertionError: expected [] to deeply equal [ 'SIGTERM' ]
  This proved the timed-out subprocess was not killed.

Fix:
- Timeout path deletes pending run, resolves timeout error, then restarts the subprocess.
- Restart marks the old subprocess as retired, spawns a fresh subprocess, and sends SIGTERM to the retired subprocess.
- stdout/stderr/exit handlers ignore retired subprocesses so stale output and expected exits cannot affect the active kernel.
- reset() now uses the same restart path.
